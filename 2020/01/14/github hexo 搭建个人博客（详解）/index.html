    Welcome to myBlog! This is my first blog.

    Quick Start

    1.准备工具

      安装 git , 注册github,安装node.js环境,安装Visual Studio Code

    1.1 安装node.js

      安装nodejs,下载.msi格式安装包进行安装。它会自动配置环境变量（无需手动配置），打开cmd窗口，执行命令node -v查看node版本，出现版本号安装成功。最新版的node在安装时同时也安装了npm,执行npm -v查看npm版本。
      修改npm全局下载的存储路径。默认情况下，我们在执行npm install -g XXXX时，下载了一个全局包，这个包的默认存放路径位C:\Users\Administrator\AppData\Roaming\npm\node_modules下，可以通过CMD指令npm root -g查看，大多数情况下并不想占用c盘空间。
    在node安装目录下下新建两个文件夹
        node_global 全局包下载存放
        node_cache node缓存
    修改路径
    在CMD窗口执行以下两条命令：

    我们更改了npm全局模块的存放路径和cache路径，此时使用下载的包时将会出现（找不到命令）错误，因为电脑系统现在还不知道你把默认路径给改了，所以需要修改系统环境变量
    新增系统变量NODE_PATH,变量值为D:\tools\nodejs\node_global\node_modules
    修改用户变量为D:\tools\nodejs\node_global
    测试一下在CMD窗口执行
    npm install -g express
    就可以看到下载包已经到指定的node_global文件夹下了。

    1.2 安装hexo

    上面的环境配置成功以后，你就可以通过cmd命令行使用npm来安装Hexo了。只需输入下面的一行命令，等待安装完成。

    npm install -g hexo-cli

    在电脑的某个地方新建一个名为hexo的文件夹（名字可以随便取），比如以我的是D:\my_hexo_blog为例，由于这个文件夹将来就作为你存放代码的地方，所以请确定好合适的位置，不要随便乱放。然后我们cmd进入该文件夹，运行下面的命令进行初始化
    hexo init
    npm install
    Hexo初始化会自动下载一些文件到这个目录
    执行hexo g 或者 hexo generate 生成静态文件到public文件夹下
    执行hexo s 或者 hexo server 启动服务
    就可以在本地打开 http://localhost:4000/ 预览了

    1.3 安装git （略）

    2.更换主题

    按Ctrl+C关闭server服务器，然后在官网 或者 https://github.com/hexojs/hexo/wiki/Themes 选择需要的主题。
    以主题为NexT（https://github.com/iissnan/hexo-theme-next ）为例
    git clone https://github.com/iissnan/hexo-theme-next themes/next

    2.1更换主题

    修改Hexo目录下的_config.yml配置文件中的theme属性，将其设置为上面的next。具体操作为，打开创建的Hexo目录下的_config.yml这个配置文件，然后找到# Extensions这个关键字，下面有theme: landscape这一行，把这一行改为theme: next，然后执行

    访问http://localhost:4000/ 预览就改变了

    3.部署到github

    3.1配置SHH keys密钥

    检查目录是否已经有一个公共SSH密钥
    $ ls -al ~/.ssh

    执行下面命令生成SSH，敲三次回车，既可以生成id_rsa.pub文件，里面就是SSH keys的内容。

    $ ssh-keygen

    配置SSH keys到GitHub
    打开文件管理器（就是平时打开我的电脑的文件夹），找到C:\Users\frank\ .ssh\id_rsa.pub这个文件，用记事本打开，复制里面的内容。

    打开你的github官网，登陆后点击右上角头像弹出下拉栏里，点击Setting进入设置，点击SSH and GPG keys设置，点击New SSH keys添加刚刚复制的SSH密钥，Title备注自己能分清楚就好。


    检验SSH是否配置好
    $ ssh -T git@github.com

    3.2创建github厂库并部署

    创建repository
    在自己Github主页右下角，创建一个新的repository。比如我的Github账号是imsofter，那么我应该创建的repository名字应该是imsofter.github.io

    部署
    编辑_config.yml，要把下面的zippera都换成你的账号名。
    deploy:
    type: git
    repository: https://github.com/binarylxz/binarylxz.github.io
    branch: master
    执行下列指令即可完成部署。据说最新版本的hexo 中，这里的 type 要写成 git，而不是 github。
    hexo clean
    hexo generate
    hexo deploy
    就可以访问了myblog

    3.常见问题

    3.1图片加载问题

    首先确认主页配置的_config.yml中：post_asset_folder:true
    然后执行

    npm install hexo-asset-image --save

    若出现错误则先执行：

    npm install -g cnpm --registry=https://registry.npm.taobao.org

    后执行：

    npm install hexo-asset-image --save

    等待一小段时间后，再运行hexo n “xxxx”来生成md博文时，/source/_posts文件夹内除了xxxx.md文件还有一个同名的文件夹。
    在xxxx.md中想引入图片时，先把图片复制到xxxx这个文件夹中
    在xxxx.md中按照markdown的格式引入图片：
    ![你想输入的替代文字](xxxx/图片名.jpg)

    然后

    本地查看图片是否加载成功 如果没有执行

    npm install https://github.com/CodeFalling/hexo-asset-image --save
    安装0.0.5版本的hexo-asset-image插件。
    使用 npm install hexo-asset-image --save 安装的是1.0.0版本的hexo-asset-image插件。 两者最直接的区别是映射关系不同。
